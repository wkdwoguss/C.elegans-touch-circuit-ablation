01 환경 구축

OpenWorm 스택은 Python 3.10을 요구하는데(owmeta 미지원 문제로 3.10 상한), 시스템 기본 Python이 3.13이라 deadsnakes PPA로 3.10을 별도 설치하고 venv를 구성했다.

  • venv: ~/wkdwoguss/openworm/venv310 — c302 0.12.0, pyNeuroML 1.3.22, libNeuroML 설치 완료 (owmeta는 미설치, c302는 이 상태로도 정상 동작)
  • Docker 이미지: openworm/openworm pull 완료
  • ffmpeg 호스트에 추가 설치 (matplotlib mp4 출력용)

02 c302 CLI 버그 2건

howto.md에 적힌 커맨드 예시를 그대로 실행하면 조용히 틀린 결과가 나온다. 둘 다 JSON 스타일 따옴표가 c302의 자체 파서와 충돌하는 동일 계열 버그다.

Bug 1 — 리스트 인자의 따옴표
-cells ["PLML","PLMR"] 처럼 따옴표를 넣으면 셀 이름에 따옴표 문자가 그대로 박혀서 Finished loading 0 cells가 된다.
→ 올바른 형식: -cells [PLML,PLMR] (따옴표 없이)
Bug 2 — paramoverride의 따옴표 (더 위험함)
-paramoverride {"unphysiological_offset_current":"2.9pA"}를 그대로 쓰면, 따옴표가 포함된 별도의 키로 저장되고 실제 unphysiological_offset_current는 기본값 0pA로 남는다. 에러 없이 조용히 무시되기 때문에, 확인 없이 진행했다면 3일 내내 자극 전류가 0이었다는 걸 몰랐을 것이다.
→ 올바른 형식: -paramoverride {unphysiological_offset_current:2.9pA}

추가로: .dat trace 파일의 컬럼 순서는 -cells 입력 순서가 아니라 셀 이름의 알파벳순이다. (4셀 스모크 테스트 때는 우연히 입력 순서와 일치해서 처음엔 놓쳤음)

03 커넥톰 검증

cect.readers.SpreadsheetDataReader (397개 뉴런)로 가설이 요구하는 연결이 실제 존재하는지 확인했다.

경로결과비고
PLM → PVC존재Gap junction + Glutamate 화학시냅스
PVC → AVB강함Glutamate, weight 5~12 (4개 조합 모두 확인)
AVB → DB1/VB1비대칭AVBR↔DB1, AVBL↔VB1 (교차배선, 알려진 생물학과 일치)
AVD → AVA강함Glutamate weight 13~19, 양방향 교차 연결
ALM → AVD예상보다 약함ALML→AVDR만 weight 1; 대신 ALM→PVC 직접 경로(weight 2) 존재

05 Dose-response 스윕

PLM 자극 전류를 0.5~5pA로 스윕한 결과, 완만한 S자 곡선이 아니라 2~2.9pA 사이에서 뚝 끊기는 계단형(all-or-none) 반응이 나타났다. IAF(integrate-and-fire) 뉴런 모델의 특성상 임계값을 넘으면 반응 크기가 곧바로 포화된다.

06 PVC Ablation 실험 & V1 영상

PVCL/PVCR을 셀 목록에서 제거한 51셀 네트워크를 동일한 PLM 자극으로 재실행했다.

핵심 발견
PLM→AVD→AVA 경로는 우회 연결 덕분에 크게 안 죽는다 (AVD +71mV, AVA +59mV — WT와 거의 동일). 반면 모터 출력 단(VA1)은 완전히 사라진다: WT +75mV → ablated 0mV. AVB/VB1은 완전 차단이 아니라 부분 감소(AVB 63mV→25mV). 즉 "PVC에서 신호가 뚝 끊긴다"는 단순 그림보다, 특정 모터 경로(후진/VA1)만 선택적으로 소실되는 더 미묘한 결과.

07 Sibernetic (V2 로코모션 영상) 진행 중

실제 꼬마선충 몸통이 움직이는 3D 물리 시뮬레이션 영상. Docker 이미지의 기본 파이프라인(master_openworm.py → sibernetic_c302.py → c302_FW.py)을 조사한 결과, 구조적 문제 3건을 발견하고 패치했다.

  1. 01
    기본 -reference FW는 PVC/PLM/AVA/AVD/ALM/AVM을 아예 포함하지 않음 (AVB+모터뉴런만) — "PVC ablation" 자체가 이 파이프라인에서는 원래 성립 불가능했다.
  2. 02
    c302_TapWithdrawal.py가 터치 서킷 셀은 포함하지만(저자 주석: "아직 개발 중, 올바른 행동 안 함"), 터치 자극 코드가 전부 주석 처리되어 있어 자극이 전혀 발생하지 않음.
  3. 03
    muscles_to_include = False로 하드코딩되어 있어 뉴런이 근육에 연결조차 안 되어 있었음 (몸통을 움직일 방법이 없음).

세 가지를 모두 패치한 c302_TouchTW_WT.py / c302_TouchTW_PVCablated.py (PLM에 실제 자극 펄스 추가 + 근육 연결 활성화 + PVC 제거) 를 만들어 컨테이너에 볼륨 마운트했다. master_openworm.py도 -reference를 환경변수로 바꿀 수 있게 패치.

시간 예산 이슈
최초 5000ms 설정으로 돌렸더니 물리 엔진(PCISPH) 로그에 "time left: 20.7시간"이 찍혔다 — CPU 전용 OpenCL(AMD APP SDK 기반, 호스트의 RTX A5000 GPU는 컨테이너에서 인식 불가) 환경이라 하루 안에 안 끝남. DURATION=500ms로 줄이고 터치 자극 타이밍도 50~350ms로 앞당겨 재시작 (예상 소요 약 2시간). 현재 openworm_wt, openworm_pvc_ablated 두 컨테이너가 이 설정으로 백그라운드 실행 중.

08 남은 일 / Day 2 계획

  • Sibernetic V2 영상 완성 확인 (진행 중, ETA 약 2시간)
  • PVC ablation을 나머지 모터뉴런(VA2-12, DB2-7, VB1-11, DA1-9)까지 확장 — 기존 데이터로 즉시 가능, 새 시뮬레이션 불필요
  • 양성 대조군: AVB 단독 ablation(전진 결손 예상), AVA 단독 ablation(후진 결손 예상)
  • Bypass control: AVB에 직접 전류 주입 — PVC 없이도 모터 응답 자체는 가능한지 확인
  • Parameter set B/C 간 재현성 확인
  • 그래프이론(NetworkX) 컨틴전시 트랙 — 병렬로 진행 가능
